关于配置hexo-fluid主题的一些坑

安装方式：

方式一

此方法安装的fluid主题所在目录如下：
E:\MyBlog\node_modules\hexo-theme-fluid
配置完之后，直接部署到github的page能用；
但是：利用 Github Actions 自动部署 Hexo 博客时，fluid的配置就不能用了
原因：node_modules文件夹没有被推送进博客源码仓库

下面来想一个解决办法

方式三
适用于自定义了一些代码，或想体验其他分支的情况，以 dev 分支为例。

确定自己的 fluid 目录已经开启 git，并且所有改动都已 commit；

新建分支步骤：

查看所有分支.

 git branch -a(查看所有分支包括本地分支和远程分支).

 git branch -r(查看远程分支).
本地创建新的分支.

git checkout -b branchname(直接新建一个分支然后切换至新创建的分支).就是创建加切换分支.

将新分支推送至GitHub.

把 fluid 仓库的 develop 分支拉取到自己当前的分支上（也可新建一个分支再拉取）：

下载fluid解压到thmes提交试试

删除主题文件夹试试
更改fluid文件夹名称试试

最终解决，不是很完美

下载 最新 release 版本 (opens new window)解压到 博客根目录下的themes 目录，并将解压出的文件夹重命名为 fluid。然后再push到博客源码仓库

但是，此方法不能把fluid地址栏的图标也自定义了
备注：（最终又可以了，说明hexo框架也存在一些BUG）
